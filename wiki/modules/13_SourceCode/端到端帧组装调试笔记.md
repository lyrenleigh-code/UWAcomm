## 1. 问题起源

P3(SC-TDE) 开发时，用户提出三个关键问题：
1. **"为什么信号没有做帧组装？"** — 模块08已有 `frame_assemble_sctde` / `frame_parse_sctde`，但测试未使用
2. **"端到端方式需要考虑帧组装，粗估计也需要用同步头"** — LFM前导是多普勒粗估计（模块10）的基础
3. **"帧组装的信号应该是实数信号，同步信号也应该是通带实数信号"** — 物理上DAC/ADC处理的是通带实信号

---

## 2. 架构演进

### V1（旧）：逐块独立处理，无帧组装

```
TX: QPSK → [CP+data] → RRC成形(基带) → 直接送信道
CH: gen_uwa_channel(单块shaped, 独立seed)
PB: upconvert → 实噪声 → downconvert
RX: RRC匹配 → 下采样 → 去CP+FFT → MMSE → BCJR
```

- 每块独立通过信道（独立seed→独立衰落）
- 无同步检测（假设已知帧起始）
- 无多普勒估计（使用已知α）
- 静态测试 BER=0% @ 3dB+

### V2（中间）：基带复数帧

尝试在基带拼接复LFM + 复数据，**但物理不正确**（帧信号是复数，无法DAC输出）。

### V3/V2.1（最终）：通带实数帧 + Turbo均衡

```
TX: QPSK → [CP+data] → 09 RRC成形(基带) → 09上变频(通带实数)
    08 gen_lfm(通带实LFM) → 功率归一化
    08帧组装: [LFM_pb | guard | data_pb | guard | LFM_pb]  ← 全实数
CH: 等效基带帧 → gen_uwa_channel(复增益×复信号) → 09上变频 → +实噪声
RX: 09下变频 → 08同步检测(基带,直达径窗口) →
    10多普勒估计(前后LFM) → 10重采样补偿(-alpha) →
    提取数据 → 09 RRC匹配 → 跨块Turbo均衡(LMMSE-IC+DD信道更新+BCJR) → 译码
```

## 3. 关键设计决策

### 3.1 LFM前导生成

| 决策 | 理由 |
|------|------|
| 使用 `gen_lfm`(模块08) 生成通带实LFM | 物理上LFM是通带实信号 `cos(2π(f_start·t + 0.5·μ·t²))` |
| 同时生成等效基带LFM `exp(j·2π·(-bw/2·t + 0.5·μ·t²))` | 信道仿真和同步检测需要基带参考 |
| LFM带宽 = 信号带宽 `sym_rate × (1+rolloff) = 8100Hz` | 匹配下变频滤波器带宽 |
| LFM频率范围: `fc ± bw/2 = 7950 ~ 16050 Hz` | 以载波为中心 |
| LFM持续时间: 50ms → 2400采样点 | TB积=405，提供足够的同步处理增益 |

### 3.2 功率归一化

```matlab
data_rms = sqrt(mean(data_pb.^2));
lfm_rms  = sqrt(mean(LFM_pb.^2));     % cos波 RMS ≈ 0.707
lfm_scale = data_rms / lfm_rms;
LFM_pb = LFM_pb * lfm_scale;
LFM_bb = LFM_bb * lfm_scale;          % 基带同步缩放
```

### 3.3 帧结构

```
[LFM_pb(2400) | guard(800) | data_pb(变长) | guard(800) | LFM_pb(2400)]
```

- 保护间隔 800采样 > 最大多径扩展 720采样 (90符号×8sps)
- 双LFM结构：前导用于同步，前后LFM用于多普勒估计

### 3.4 信道仿真域

| 层次 | 域 | 原因 |
|------|-----|------|
| 帧组装 | 通带实数 | 物理正确，DAC接口 |
| 信道施加 | 等效基带复数 | 复增益×复信号=正确 |
| 噪声添加 | 通带实数 | `upconvert(rx_bb) + real_noise` |
| 同步检测 | 基带复数 | 避免载波相位不敏感 |

## 4. BUG发现与修复记录（按时间顺序）

### BUG-1：alpha_est空值崩溃
- **现象**：`alpha_est_matrix(fi,si) = alpha_est` 赋值错误，alpha_est为1×0空数组
- **根因**：`est_doppler_xcorr` 内部 `abs(xcorr)` 全局搜索命中LFM2 → 搜索第二个LFM越界返回空
- **修复**：`if ~isempty(alpha_tmp) && isfinite(alpha_tmp) ... else alpha_est = dop_rate`

### BUG-2：同步误检LFM2 → 数据提取越界
- **现象**：`sync_pos2=53181`，`ds=56381` 完全越界，BER≈50%
- **根因**：Jakes衰落使LFM2相关峰 > LFM1，`0.7×max` 阈值排除了LFM1
- **修复**：前半段搜索 → 直达径窗口搜索（见BUG-5）

### BUG-3：Doppler补偿方向错误（关键BUG）
- **现象**：`dop-only`（静态信道+微小Doppler）BER≈45%
- **根因**：`comp_resample_spline(bb, +alpha)` 使用 `pos = (1:N)*(1+α)`，但信道压缩后应**除以(1+α)**
  - 正确补偿方向：`pos = (1:N)*(1-α) ≈ (1:N)/(1+α)`
  - 传`+alpha`导致Doppler翻倍而非消除
  - V1短块(~3000样本)误差=N·α²≈0.02样本→可忽略
  - V2长帧(~55000样本)误差=2·N·α≈9样本→超过一个符号周期
- **修复**：`comp_resample_spline(bb_raw, -dop_rate, fs, 'fast')` — 取负

### BUG-4：信道seed依赖SNR索引
- **现象**：不同SNR点BER不单调（有的0%有的50%）
- **根因**：`'seed', 200+fi*100+si` → 每个SNR看不同信道实现
- **修复**：`'seed', 200+fi*100` → 信道固定，只变噪声

### BUG-5：sync检测反射径 → oracle H_est时延不匹配（关键BUG）
- **现象**：`sync=121`（第3径delay=120），oracle用绝对时延[0,5,15,40,60,90]
- **根因**：数据以第3径为定时参考，有效时延应为 `mod(delays-15, N)=[241,246,0,25,45,75]`
  - 时延完全不匹配 → MMSE反转错误信道 → BER≈50%
- **修复**（三步）：
  1. **有效时延偏移**：`eff_delays = mod(sym_delays - sync_offset_sym, blk_fft)`
  2. **分数相位斜坡**：`phase_ramp = exp(+j·2π·frac·k/N)` 补偿残余分数符号偏移
  3. **直达径窗口**：sync搜索限制在前50样本内，确保找到直达径/最近反射径

### BUG-6：噪声影响sync位置（同一信道不同SNR结果不一致）
- **现象**：固定信道seed，fd=5Hz在5dB=14%但10dB=49%
- **根因**：sync在含噪信号上做 → 不同噪声种子导致sync位置变化
- **修复**：**sync在无噪声信号上做一次**(per fading config)，SNR循环内只变噪声

## 6. 关键技术方案

### 6.1 同步策略（最终版）

```matlab
% 无噪声信号上做一次同步（per fading config）
[bb_clean,~] = downconvert(rx_pb_clean, fs, fc, bw_lfm);  % 无噪声
bb_clean_comp = comp_resample_spline(bb_clean, -dop_rate, fs, 'fast');
[~, ~, corr_clean] = sync_detect(bb_clean_comp, LFM_bb_n, 0.3);
% 直达径窗口：只搜前50样本（~6符号），确保找直达径而非远处反射径
dw = min(50, round(length(corr_clean)/2));
[sync_peak, sync_pos_fixed] = max(corr_clean(1:dw));
sync_offset_sym = round((sync_pos_fixed - 1) / sps);
```

### 6.2 Oracle H_est（含sync偏移补偿）

```matlab
% 整数循环移位 + 分数相位斜坡
sync_offset_sym_frac = sync_offset_samp/sps - sync_offset_sym;
phase_ramp_frac = exp(+1j*2*pi*sync_offset_sym_frac*(0:blk_fft-1)/blk_fft);

eff_delays = mod(sym_delays - sync_offset_sym, blk_fft);
h_td = zeros(1, blk_fft);
for p=1:length(sym_delays), h_td(eff_delays(p)+1) = h_norm(p); end
H_est = fft(h_td) .* phase_ramp_frac;
```

### 6.3 跨块Turbo均衡 + DD信道更新

```
for titer = 1:6
    % 1. Per-block LMMSE-IC（用当前H_est）→ LLR
    % 2. 跨块BCJR译码
    % 3. soft_mapper → 各块软先验 x_bar
    % 4. DD信道重估计（iter≥2且置信度够时）:
    %    H_dd = Y·X̄*/(|X̄|²+ε) → 稀疏投影(有效时延) → 新H_est
end
```

### 6.4 Doppler补偿方向

```matlab
% gen_uwa_channel: 正alpha → 时间压缩 (scale = 1/(1+α))
% comp_resample_spline: pos = (1:N)*(1+alpha) → 拉伸
% 正确补偿：传 -alpha → pos = (1:N)*(1-α) ≈ (1:N)/(1+α)
bb_comp = comp_resample_spline(bb_raw, -dop_rate, fs, 'fast');
```

## 9. 时变信道估计开发（2026-04-07）

### 新增4个函数

| 函数 | 类型 | NMSE(fd=5Hz) | 状态 |
|------|------|-------------|------|
| `ch_est_bem` V1 | CE/P/DCT-BEM基扩展 | **-15.9dB(CE)** | ✅ 最优 |
| `ch_track_kalman` V1 | 稀疏Kalman AR(1) | 调优中 | ✅ K参数重设计 |
| `ch_est_tsbl` V2 | T-SBL时序稀疏贝叶斯 | 检测23/6径 | ✅ 需继续调参 |
| `ch_est_sage` V1 | SAGE/EM多径参数估计 | 时延完美匹配 | ✅ |

### BEM为什么最优
- **批处理**：一次用850+导频观测，噪声被平均
- **参数化**：h(n)=Σ c_q·b_q(n)，Q=9阶CE-BEM只需54个系数
- **先验**：只需fd(多普勒频率)和σ²(噪声方差)

### Kalman为什么差
- **逐符号处理**：每步只有1个观测
- **旧参数**：α=J₀(2πfd/fs)≈0.99999→K_gain≈0.00009→几乎不更新
- **新参数**：α=0.5^(4fd/fs)≈0.998, K_target=5%→待验证

### BEM+散布导频方案（已验证）

```
帧: [训练500|导频50|数据256|导频50|数据256|...|数据N|尾导频50]
BEM: CE-BEM Q=max(5, 2⌈fd·T⌉+3), MMSE-LS估计
均衡: 分块LMMSE-IC + 跨块Turbo BCJR
```
