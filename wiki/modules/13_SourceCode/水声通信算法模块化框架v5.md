# 水声通信算法模块化框架 v5.0

> 11个编号模块 + PTR被动时反转 + Turbo外信息迭代
> 覆盖 SC-TDE / SC-FDE / DSSS / OFDM / OTFS / FH-MFSK + 阵列接收

**版本演进**：
- **v1.0** 模块1~9
- **v2.0** 模块10/11/12
- **v3.0** 模块10拆分 · 迭代回环 · 6'调序
- **v4.0** PTR · RLS-DFE(+PLL) · LLR↔符号接口
- **v5.0** Turbo迭代重构: SISO均衡+SISO译码+外信息交换 · MMSE-IC · 交织纳入迭代环

## v5.0 框架变更一览（相对v4.0）

| 类型 | 变更内容 |
|:---:|---|
| ☆ | Turbo迭代架构重构：遵循外信息(Extrinsic Information)交换原则，模块间只传递 L_e，避免信息自我强化 |
| ✎ | 迭代回环重新定义：7'(SISO均衡) → 3'(解交织) → 2'(SISO译码) → 3(交织) → 7'，交织器纳入迭代环路 |

| ✎ | 删除"须配合系统码"限制：非系统卷积码+BCJR完全支持Turbo均衡 |

## 二、各通信体制的模块选用

### 1. 单载波时域均衡 (SC-TDE) `v5 Turbo外信息迭代`

> 跳过模块5、6；接收端PTR空间聚焦 → 时域自适应均衡（RLS/DFE）；SISO(BCJR)译码器输出外信息；交织器纳入迭代环路；Turbo迭代3~6次

**发射链路**：
`1 信源编码` → `2 信道编码` → `3 交织` → `4 符号映射` → `7 导频插入` → `8 同步+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列(可选)` → `10-1 粗多普勒` → `PTR 被动时反转` → `7' LS信道估计+RLS-DFE+PLL` → `10-2 PLL(嵌入)` → `4' LLR判决` → `3' 解交织` → `2' 信道解码` → `1' 信源解码`

**↻ Turbo 外信息迭代流程（v5重构）**：
- **初始化**：x̄=0, σ²_x=1, L_a,eq=0
- **每次迭代**：PTR → RLS/DFE均衡+软ISI消除 → `soft_demapper`(L_e,eq=LLR-L_a) → 解交织 → SISO译码(`siso_decode_conv.m`) → L_e,dec+后验 → 交织 → `soft_mapper`(x̄,σ²_x) → 下一轮
- **核心原则**：模块间只传递外信息 L_e = L_post - L_prior，避免信息自我强化导致发散
- **终止条件**：最大迭代次数(3~6次) 或 CRC校验通过

---

### 2. 单载波频域均衡 (SC-FDE) `v5 迭代MMSE-IC`

> 跳过模块5、6；发射端分块加CP；接收端PTR(可选) → 分块FFT → 迭代MMSE-IC均衡(Tuchler公式) → IFFT；SISO译码器外信息反馈；Turbo迭代3~6次

**发射链路**：
`1 信源编码` → `2 卷积/Turbo编码` → `3 交织` → `4 QPSK/QAM映射` → `7 分块+CP+前/后导码` → `8 同步+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列(可选)` → `10-1 复自相关→重采样` → `6' 去CP+分块FFT` → `PTR(可选)` → ↻`7' MMSE均衡→IFFT` → `10-2 CFO校正` → `4' LLR` → `3' 解交织` → `2' 信道解码`↻ → `1' 信源解码`

**↻ Turbo迭代（3~6次）**：
7'(`eq_mmse_ic_fde`, W=H*/(‖H‖²+σ²/σ²_x)) → `soft_demapper`(L_e=4μ/σ²·x̃-L_a) → 3' → 2'(SISO-BCJR) → 3 → `soft_mapper`(x̄,σ²_x) → 7'

---

### 3. 直接序列扩频 (DSSS / DS-CDMA)

> 启用模块5(DSSS直扩)，跳过模块6；无迭代，单次直通

**发射链路**：
`1 信源编码` → `2 信道编码` → `3 交织` → `4 符号映射` → `5 DSSS扩频` → `7 导频插入` → `8 同步+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列预处理(可选)` → `10-1 粗多普勒补偿` → `7' 导频提取→估计&均衡` → `10-2 残余相位校正` → `5' 相关解扩` → `4' 符号判决` → `3' 解交织` → `2' 信道解码` → `1' 信源解码`

---

### 4. 正交频分复用 (OFDM)

> 启用模块6(IFFT/FFT+CP)，跳过模块5；10-1用CP自相关估计+重采样；6'去CP+FFT；无迭代，单次直通

**发射链路**：
`1 信源编码` → `2 信道编码` → `3 交织` → `4 QAM映射` → `6 IFFT+CP插入` → `7 频域导频插入` → `8 前导码+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列预处理(可选)` → `10-1 CP自相关→重采样` → `6' 去CP+FFT` → `7' 导频→LS估计→FDE均衡` → `10-2 残余CFO旋转校正` → `4' 符号判决` → `3' 解交织` → `2' 信道解码` → `1' 信源解码`

---

### 5. 正交时频空调制 (OTFS)

> 模块6变体：ISFFT+Heisenberg（发）/ Wigner+SFFT（收）；模块7：DD域嵌入单脉冲导频+保护区；10-1可弱化；MP均衡器必选迭代

**发射链路**：
`1 信源编码` → `2 信道编码` → `3 交织` → `4 QAM映射` → `6 ISFFT+Heisenberg(N×M格点)` → `7 DD域嵌入导频+保护区` → `8 整帧CP+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列(可选)` → `10-1 粗对齐(可弱化)` → `6' Wigner+SFFT→DD域` → ↻`7' MP均衡(BP)` → `4' LLR` → `3' 解交织` → `2' 信道解码`↻ → `1' 信源解码`

**↻ 双层迭代**：外层Turbo 3~5次 (7'→3'→2'(SISO)→3→7' 外信息交换) × 内层MP 10~30次(DD域BP)

---

### 6. 跳频多频移键控 (FH-MFSK)

> 模块4变体(MFSK) + 模块5变体(跳频)，跳过模块6；无迭代，单次直通

**发射链路**：
`1 信源编码` → `2 信道编码` → `3 交织` → `4 MFSK映射` → `5 FH跳频` → `7 导频插入` → `8 同步+帧组装` → `9 成形/上变频` → **信道**

**接收链路**：
`9' 滤波/下变频` → `8' 同步+帧解析` → `11 阵列预处理(可选)` → `10-1 粗多普勒补偿` → `7' 导频提取→估计&均衡` → `10-2 残余相位校正` → `5' 解跳频` → `4' 能量检测` → `3' 解交织` → `2' 信道解码` → `1' 信源解码`

---

### 7. 阵列波束形成增强接收（叠加配置）

> 不独立成体制，而是作为模块11前处理叠加在上述任意体制之上；输出单路高质量复基带信号，下游流程无需改动

**模式A（精度优先）**：
`M路ADC采样 R[m,n]` → `时延标定 bf_delay_calibration` → `空时变采样排序+三次样条重建` → `等效采样率 ≈ M·fs` → `高精度多普勒估计（CRLB降低M²倍）` → 进入标准接收流程

**模式B（SNR优先）**：
`M路ADC采样 R[m,n]` → `DAS波束形成（时延对齐+相位补偿）` → `SNR提升 ≈ 10·log₁₀(M) dB` → `标准多普勒估计（低SNR场景适用）` → 进入标准接收流程

## 四、框架适配说明与关键调整点

### ☆ Turbo 均衡核心原则与新增组件 (v5重构)

- **外信息原则**：L_e = L_post - L_prior，每个SISO模块只输出新增信息，避免自我强化导致发散
- **迭代环路**：7'(SISO均衡) → 3'(解交织) → 2'(SISO-BCJR) → 3(交织) → 7'
- **新增 `eq_mmse_ic_fde.m`**：迭代MMSE-IC，W[k]=H*/(‖H‖²+σ²_w/σ²_x)，IC: Ỹ=Y-(1-WH)X̄
- **新增 `soft_demapper.m`**：均衡输出→编码比特外信息，L_e=4μ/σ²·x̃-L_a
- **新增 `soft_mapper.m`**：后验LLR→软符号x̄+残余方差σ²_x
- **改造 `siso_decode_conv.m`**：新增编码比特后验LLR输出（第3输出参数）
- **SC-TDE**：保留RLS/DFE+软ISI消除，但译码器切SISO，反馈为外信息
- 终止：最大迭代次数(3~6次) 或 CRC通过

### ⌂ OTFS 必须从 OFDM 拆分的原因

- 模块6完全不同：OFDM 用 IFFT/FFT，OTFS 用 ISFFT+Heisenberg（发）/ Wigner+SFFT（收）
- 模块7完全不同：OFDM 频域子载波导频 vs OTFS DD域嵌入单脉冲导频+保护区
- 模块8有差异：OTFS 用整帧 CP（长度≥τ_max·fs），而非每符号 CP
- 均衡器完全不同：OTFS 估计稀疏路径 {h_i, l_i, k_i}，使用 MP 均衡器（迭代回环）
- 多普勒处理方式根本不同：OTFS 在 DD 域直接分辨路径多普勒，模块10-1可弱化
- 对应 MATLAB 文件：`gen_otfs_signal.m`、`ch_est_otfs.m`、`mp_detector.m`

### + 模块11 阵列预处理的定位与约束

- 插入位置：8'（同步）之后，10-1（粗多普勒）之前；对下游完全透明
- 模式A（精度优先）：空时变采样重建，等效采样率提升至 M·fs，CRLB 降低 M² 倍
- 模式B（SNR优先）：DAS波束形成，SNR 提升约 10log₁₀(M) dB
- 精度要求：阵元时延标定误差须 < Ts/(2M)，否则等效采样率提升失效
- MATLAB：`bf_delay_calibration.m`、`bf_nonuniform_resample.m`、`bf_conventional.m`

### ✎ Turbo迭代环路：7'(SISO-EQ) ↔ 3'/3(交织) ↔ 2'(SISO-DEC) (v5重构)

- **环路结构**：7'(SISO均衡,输出L_e,eq) → 3'(解交织) → 2'(SISO译码,输出L_e,dec) → 3(交织) → 7' | 交织器是环路必要组成部分
- **10-2位置**：残余多普勒补偿在Turbo迭代前一次性完成，不参与迭代环路（v5调整）
- **SC-TDE Turbo**：7'=RLS/DFE+软ISI消除 + soft_demapper → 3' → 2'=SISO(BCJR) → 3 + soft_mapper → 7'，3~6次
- **SC-FDE Turbo**：7'=迭代MMSE-IC(`eq_mmse_ic_fde`) + soft_demapper → 3' → 2'=SISO(BCJR) → 3 + soft_mapper → 7'，3~6次
- **OTFS**：外层Turbo 3~5次(7'→3'→2'→3→7') × 内层MP(DD域BP) 10~30次
- **DSSS / OFDM(基础) / FH-MFSK**：无迭代，单次直通
- Turbo均衡适用于任何卷积码（含非系统码），译码器使用BCJR/MAP算法
- MATLAB调度器：`turbo_equalizer_scfde.m`、`turbo_equalizer_sctde.m`、`turbo_equalizer_otfs.m`

### ⌂ 模块10拆分为10-1(粗)和10-2(残余)的原因

- **10-1 粗多普勒补偿**：位于6'之前，做宽带重采样去除整体时间压缩/扩展，确保后续FFT/逆变换不受频偏影响
- **10-2 残余多普勒补偿**：位于7'之后，做残余CFO相位旋转校正或PLL载波跟踪，基于信道估计结果精修
- 两者物理含义不同：10-1处理宽带多普勒(时间压缩)，10-2处理窄带残余(频率偏移/相位旋转)
- 位置逻辑：粗补偿必须在逆变换前（否则FFT子载波错位），残余补偿在均衡后（利用信道估计信息）
- 同属文件夹 `10_DopplerProc/`：`doppler_coarse_est.m`、`doppler_resample.m`、`doppler_residual_cfo.m`、`doppler_pll_track.m`
- SC-TDE特殊：10-2的PLL嵌入均衡器内部，与7'紧耦合而非独立调用

### ⚠ SC-TDE 与 SC-FDE 的关键区别

- 发射端完全相同，区别仅在接收端均衡器结构
- SC-TDE (v4)：PTR → 7'为时域RLS/DFE(+PLL)，第1次线性RLS，第2+次DFE+软反馈，无FFT
- SC-FDE：7' 为分块FFT→MMSE→IFFT，10-2独立做残余CFO校正，可选Turbo迭代3~6次
- SC-FDE 的多普勒估计专用前/后导码占用帧开销，SC-TDE 无此需求
- 10-1均使用粗多普勒重采样，但SC-FDE更依赖复自相关幅相联合算法
- 参数文件应分开：`sys_params_sctde.m` vs `sys_params_scfde.m`

### ⚠ OTFS 特殊实现约束

- CP 必须是整帧 CP，须覆盖所有时延 + 保护
- 保护区尺寸：guard_l ≥ ⌈τ_max·M·Δf⌉，guard_k ≥ ⌈|α|_max·fc·N·T_sym⌉+2
- 分数多普勒（k_i 非整数）时需加发射端窗函数或扩展虚拟格点
- MP 均衡器复杂度 O(P·NM·|X|)，需评估实时性；可用带阻尼的迭代版本提升稳定性
- 10-1可弱化：DD域天然分辨多普勒，粗对齐仅做时间同步校准即可
- `get_data_indices.m` 须自行实现（返回去掉导频+保护区的数据格点索引）
