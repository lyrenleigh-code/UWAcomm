# 模块02 — 信道编解码 (ChannelCoding)

> 文件夹：`02_ChannelCoding/src/Matlab/`
> 框架位置：发射端第2步(编码) / 接收端倒数第2步(解码)
> 所有体制均使用；v5中 `siso_decode_conv.m` 为Turbo均衡核心组件

---

## 功能定位

- **发射端**：添加冗余校验比特，使接收端能纠正传输错误
- **接收端**：利用冗余信息检测并纠正错误
- **Turbo均衡**：SISO(BCJR)译码器输出外信息 L_e，参与迭代环路

## 编码方案详情

### Hamming码
- 分组码 (n, k) = (2^r-1, 2^r-1-r)
- 最小汉明距 d_min = 3，可纠正1位错误
- 简单高效，适合低误码场景

### 卷积码
- 码率 R = 1/2，约束长度 K = 7
- 生成多项式：[171, 133]（八进制）= NASA标准
- 编码记忆长度 m = K-1 = 6
- 自由距 d_free = 10

### Turbo码
- 双RSC并行级联，K=4
- 反馈多项式=15(八进制)，前馈多项式=13(八进制)
- 码率 R = 1/3（含尾比特）
- 组件间使用随机交织器（调用模块03）

### LDPC码
- Gallager随机构造稀疏校验矩阵
- Min-Sum BP迭代译码
- 支持收敛追踪和提前终止

### SISO译码器（v5关键）
- **BCJR/MAP算法**：前向α、后向β递推
- **3个输出**：
  1. 信息比特外信息 L_e,info
  2. 信息比特后验LLR L_post,info
  3. **编码比特后验LLR L_post,coded**（v5新增，用于Turbo均衡反馈）
- 外信息原则：L_e = L_post - L_prior
- 支持任何卷积码（含非系统码）

## 接口约定

```matlab
% 卷积编码/Viterbi解码
coded_bits = conv_encode(info_bits, trellis)
decoded_bits = viterbi_decode(received_llr, trellis, 'soft')

% Turbo编码/解码
[coded_bits, trellis] = turbo_encode(info_bits, interleaver)
[decoded_bits, L_ext] = turbo_decode(received_llr, trellis, interleaver, num_iter)

% SISO译码器（Turbo均衡用）
[L_e_info, L_post_info, L_post_coded] = siso_decode_conv(L_a, trellis)
%   L_a         : 先验LLR（来自均衡器经解交织）
%   L_e_info    : 信息比特外信息
%   L_post_info : 信息比特后验（最终判决用）
%   L_post_coded: 编码比特后验（反馈给均衡器，经交织后）
```

## 与其他模块的关系

```
[01 信源编码] → [02 信道编码] → [03 交织] → ...

Turbo迭代环路：
  ... → [03' 解交织] → [02' SISO译码 (siso_decode_conv)] → L_e,dec
  → [03 交织] → soft_mapper → [07' SISO均衡] → ...
```

- **上游**：接收模块01输出的比特流
- **下游**：编码后送入模块03交织
- **Turbo均衡**：`siso_decode_conv.m` 与模块07的 `soft_demapper/soft_mapper` 及模块03交织器形成闭环

## 各体制编码选择

| 体制 | 推荐编码 | 说明 |
|---|---|---|
| SC-TDE | 卷积码 + SISO(BCJR) | Turbo均衡必选BCJR |
| SC-FDE | 卷积码/Turbo码 + SISO(BCJR) | 迭代MMSE-IC |
| DSSS | 卷积码 + Viterbi | 单次直通，无迭代 |
| OFDM | 卷积码/LDPC + Viterbi/BP | 单次直通 |
| OTFS | 卷积码 + SISO(BCJR) | 双层迭代 |
| FH-MFSK | 卷积码 + Viterbi | 单次直通 |
