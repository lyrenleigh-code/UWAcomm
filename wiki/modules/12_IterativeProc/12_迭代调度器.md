# 模块12 — 迭代调度器 (IterativeProc)

> 文件夹：`12_IterativeProc/`
> 框架位置：接收端，协调 7'(SISO均衡) ↔ 3'/3(交织) ↔ 2'(SISO译码) 的迭代环路
> SC-TDE / SC-FDE 使用Turbo迭代；OTFS 使用双层迭代；DSSS/OFDM(基础)/FH-MFSK 无迭代

---

## 功能定位

- **调度Turbo迭代**：控制均衡器与译码器之间的外信息交换
- **核心原则**：L_e = L_post - L_prior，模块间只传递外信息，避免信息自我强化
- **不是独立的信号处理模块**，而是协调模块02'、03'、03、07'的迭代流程控制器

## Turbo均衡核心原理

### 外信息交换

```
SISO均衡器（模块07'）           SISO译码器（模块02'）
    ↓ 输出: L_e,eq               ↓ 输出: L_e,dec
    ↓ = L_post,eq - L_a,eq       ↓ = L_post,dec - L_a,dec
    ↓                             ↓
  [解交织 03']                  [交织 03]
    ↓                             ↓
    → L_a,dec = 解交织(L_e,eq)    → L_a,eq = 交织(L_e,dec)
```

**关键**：每个SISO模块只输出"新增信息"（外信息），去除了已知的先验信息，防止信息在循环中自我强化导致发散。

### 迭代收敛

- 每次迭代，外信息的"新信息量"递减
- 收敛标志：BER不再下降 或 CRC校验通过
- 典型3~6次迭代即可收敛

## SC-FDE / OFDM Turbo迭代流程（v5）

```
初始化:
  x̄ = 0           % 软符号（全零）
  σ²_x = 1        % 残余方差（最大不确定性）
  La_eq = 0        % 均衡器先验LLR（无先验）

for iter = 1 : K_max
  %--- SISO均衡 (模块07') ---
  [x̃, μ, σ²ñ] = eq_mmse_ic_fde(Y, H, x̄, σ²_x, σ²w)

  %--- 软解映射: 均衡输出 → 外信息LLR ---
  Le_eq = soft_demapper(x̃, μ, σ²ñ, La_eq)    % L_e = f(x̃,μ,σ²) - L_a

  %--- 解交织 (模块03') ---
  Le_deint = deinterleave(Le_eq)

  %--- SISO译码 (模块02') ---
  [L_e_info, L_post_info, L_post_coded] = siso_decode_conv(Le_deint, trellis)

  %--- 交织反馈 (模块03) ---
  L_post_int = interleave(L_post_coded)

  %--- 软映射: 后验LLR → 软符号 ---
  [x̄, σ²_x] = soft_mapper(L_post_int)

  %--- 更新均衡器先验 ---
  La_eq = clip(L_post_int - Le_eq, [-8, +8])   % 后验 - 外信息 = 新先验

  %--- 检查终止条件 ---
  if crc_pass(L_post_info), break; end
end

% 最终硬判决
decoded_bits = (L_post_info > 0);
```

### MMSE-IC权重自适应

```
W[k] = H*[k] / (|H[k]|² + σ²_w / σ²_x)
```

- 第1次迭代：σ²_x = 1（无先验），W退化为标准MMSE
- 后续迭代：σ²_x → 0（判决越确定），W → H*/(|H|²) 趋近ZF + 更准IC
- 干扰消除：Ỹ = Y - (1-WH)·X̄

## SC-TDE Turbo迭代流程

与SC-FDE类似，但均衡器不同：

```
for iter = 1 : K_max
  %--- PTR空间聚焦 ---
  y_ptr = eq_ptrm(rx_array, h_est)

  %--- 时域自适应均衡 ---
  if iter == 1
    x̃ = eq_linear_rls(y_ptr, training, params)    % 第1次：线性RLS
  else
    x̃ = eq_dfe(y_ptr, training, x̄, params)        % 第2+次：DFE+软ISI消除+PLL
  end

  %--- 后续同SC-FDE ---
  Le_eq = soft_demapper(x̃, μ, σ²ñ, La_eq)
  ... (解交织 → SISO译码 → 交织 → soft_mapper → 更新先验)
end
```

**特殊点**：
- PLL嵌入在eq_dfe内部（10-2不独立）
- 第1次用线性RLS（无先验可用于DFE反馈）
- 第2+次用DFE + 软符号反馈做ISI消除

## OTFS 双层迭代流程

```
for outer_iter = 1 : K_outer      % 外层Turbo: 3~5次

  for inner_iter = 1 : K_inner    % 内层MP: 10~30次
    %--- DD域消息传递(BP) ---
    x̃ = eq_otfs_mp(Y_dd, H_dd, x̄, params)
  end

  %--- 软解映射 + 解交织 + SISO译码 + 交织 + 软映射 ---
  ... (同SC-FDE外层)

end
```

**双层结构**：
- 内层：DD域BP消息传递均衡（利用DD域信道稀疏性）
- 外层：Turbo外信息交换（均衡器 ↔ 译码器）
- 复杂度：O(P·NM·|X|) × K_inner × K_outer

## 各体制迭代配置

| 体制 | 迭代类型 | 均衡器 | 译码器 | 迭代次数 |
|---|---|---|---|---|
| SC-TDE | Turbo | RLS-DFE+软IC | SISO(BCJR) | 3~6 |
| SC-FDE | Turbo | MMSE-IC(Tuchler) | SISO(BCJR) | 3~6 |
| OFDM(增强) | Turbo | MMSE-IC | SISO(BCJR) | 3~6 |
| OTFS | 双层 | MP(BP) + Turbo | SISO(BCJR) | 外3~5 × 内10~30 |
| DSSS | **无** | — | Viterbi | 单次直通 |
| OFDM(基础) | **无** | — | Viterbi/LDPC-BP | 单次直通 |
| FH-MFSK | **无** | — | Viterbi | 单次直通 |

## 与其他模块的关系

```
接收链路：
  ... → [10-2 残余多普勒(一次性)] → [12 迭代调度器]
                                        ├→ 07'(SISO均衡)
                                        ├→ 03'(解交织)
                                        ├→ 02'(SISO译码)
                                        ├→ 03(交织反馈)
                                        └→ ... → 最终判决
  → [05' 解扩(可选)] → [04' 符号判决] → [03' 解交织] → [02' 硬解码] → [01' 信源解码]
```

- 迭代环路内的模块：07'、03'、02'、03
- 迭代结束后：走标准解调链路（最终硬判决）
- v5调整：10-2移出迭代环，只执行一次

## 备注

- Turbo均衡是水声通信接收机性能提升的关键技术
- 典型增益：SC-FDE Turbo 3次迭代可获得2~4dB的BER改善
- LLR裁剪(clip ±8)防止数值溢出和过度自信
- CRC提前终止可节省50%以上的平均计算量
- 非系统卷积码+BCJR完全支持Turbo均衡（v5已删除系统码限制）
- EXIT chart可用于分析迭代收敛行为和调制/编码方案匹配
