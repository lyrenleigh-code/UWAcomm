# 14 流式仿真框架

> 返回 [[index]] | [[uwacomm-moc]]

## 目的

模拟真实水声通信部署：文本输入 → wav 文件传输 → 信道模拟 → wav 文件接收 → 文本输出。

## 与批处理测试（13_SourceCode）的区别

| 维度 | 13_SourceCode/tests | 14_Streaming |
|------|---------------------|--------------|
| 输入 | 随机 bits（数组） | 用户文本（UTF-8） |
| 传输介质 | in-memory 数组 | wav 文件 |
| 信道 | 同进程调用 gen_uwa_channel | 独立 channel daemon 进程 |
| 帧定位 | 已知偏移 | 滑动匹配滤波发现 |
| 多帧 | 否 | 是（长文本分帧） |
| 体制切换 | 手动配置 | AMC 自适应 |
| 并发 | 串行脚本 | 三进程并发 |
| 自组网 | 无概念 | 帧头留节点地址字段 |

## 核心概念

### 会话目录（方案 B）

每次通信任务创建 `session_<timestamp>/`，三进程通过该目录协作：
- TX 写 `raw_frames/NNNN.wav` + `NNNN.ready`（close 后原子创建）
- Channel daemon 读 raw_frames，写 `channel_frames/NNNN.wav` + `NNNN.ready`
- RX 读 channel_frames，写 `rx_out/NNNN.meta.json` + `session_text.log`

每帧独立 wav 文件避免 Windows 文件锁冲突。

### 帧协议

物理帧 = [HFM+|guard|HFM-|guard|LFM1|guard|LFM2|guard|Header|Payload]

- Header (16B) 用 FH-MFSK 调制，最鲁棒，保证控制信息永远可解
- Payload 按 Header.scheme 字段指定的体制调制
- 每段独立 CRC16 校验

### 统一 modem API

6 体制包装为 `modem_encode(bits, scheme, sys)` / `modem_decode(rx_pb, scheme, sys, meta)`，通过 `scheme` 字段分发。

### AMC（物理层指标）

RX 估计链路质量（LFM sync_peak / SNR / delay_spread / doppler）→ 决策表 → 推荐体制 → 通过 ACK 反馈 TX → TX 切换下一帧 scheme。

### 多帧协议（P1 帧结构不变，激活 idx + last_frame）

```
[Frame1: HFM+|...|Header(idx=1)|Payload1]
[Frame2: HFM+|...|Header(idx=2)|Payload2]
...
[FrameN: HFM+|...|Header(idx=N, flags.bit0=last)|PayloadN]
```

帧间无 gap（HFM+ 检测足够分辨）；末帧 `flags.bit0=1` 标记。

### 流式检测算法（hybrid V1.1）

1. **Stage A**：在 [1, 30%·frame_len + N_template] 窗口取**绝对最大**作为 frame 1 锚点（不依赖阈值）
2. **Stage B**：用 frame_len 预测后续帧位置 = `first + (i-1)·frame_len`，±5% frame_len 窗口取本地最大；预测窗口阈值降一半，单帧深衰落漏检不连锁

### FH-MFSK 软判决 LLR（V1.1）

每符号 M 个频率能量 e_freqs[0..M-1]，对每位 j：
```
LLR(b_j) = (max_e_b1 - max_e_b0) / median(e_freqs)
```

替代硬判决 ±10，给 Viterbi 真实置信度，对衰落鲁棒。

### 增量改进与坑

- **downconvert LPF 暖机**：必须预填零给 LPF 消化瞬态，否则 frame 1 HFM 被破坏
- **frame 1 阈值漏检**：Jakes 让 frame 1 peak 远低于 peak_max，纯阈值过滤会漏；改在 frame 1 预期窗口取绝对最大
- **深衰减 preset 调整**：原 1.5ms 延时展宽（75% 符号）造成严重 ISI；改 0.6ms 后可承受
- **uitable 数据格式**：cell 数组，列宽用混合数字+'1x'

## P3.1 实施记录（2026-04-15 完成）

### 产出

- `common/modem_dispatch.m` — 按 scheme 大写去连字符（'FHMFSK'/'SCFDE'/...）分发
- `common/modem_encode.m` / `common/modem_decode.m` — 薄包装入口，统一 `info` 4 字段（estimated_snr/ber/turbo_iter/convergence_flag）
- `tx/modem_encode_scfde.m` + `rx/modem_decode_scfde.m` — 从 13_SourceCode/tests/SC-FDE 抽取
- `rx/modem_decode_fhmfsk.m` — 补齐统一 info 字段
- `tests/test_p3_unified_modem.m` — 双体制回归

### 关键设计

- **modem 边界 = body only**：encoder 输出基带 body（不含 HFM/LFM），由 `assemble_physical_frame` 拼前导
- **doppler 补偿在外层**：modem_decode 假设输入已对齐（streaming 场景由 frame_detector + detect_lfm_start 处理）
- **SC-FDE 符号定时**：通过 `meta.pilot_sym`（TX 已知首 10 符号）传递，避免盲搜（streaming 场景 meta 随 wav 下发，合理）
- **fading_type 切换**：`sys.scfde.fading_type='static'` → GAMP；`'slow'` → BEM(DCT)
- **noise_var 注入**：通过 `meta.noise_var` 由外层传入，未传则用末尾 CP 段方差粗估

### 验收

| scheme  | 5dB | 10dB | 15dB | 备线 |
|---------|-----|------|------|------|
| FH-MFSK | 0%  | 0%   | 0%   | <0.5%@10dB ✓ |
| SC-FDE  | 0%  | 0%   | 0%   | <0.5%@15dB ✓ |

测试场景：基带 6 径静态多径 + 复 AWGN（bypass passband，验证 modem 本身正确性）。
完整 passband 链路（HFM/LFM 同步 + 上下变频）回归留待 P4 流水线整合。

### P3.1 demo UI（V2 流式版）

`ui/p3_demo_ui.m` — "软件无线电"流式 demo，单进程 timer 模拟连续接收。

工作流：先打开 **RX 监听开关** → 配置 TX 参数（文本默认可改 + SNR + Doppler + 衰落 + 预设 + scheme 特定参数）
→ 点 Transmit → TX 切片以 50ms/100ms (2×) 速率推入 FIFO → 实时通带示波器持续刷新 → 累积满帧后自动解调。

7 个 tab（V3.0 重构）：
1. 实时通带示波器（passband real，最近 0.4s，timer 100ms 刷新）
2. 通带频谱（标 ±fc）
3. 均衡前星座（SC-FDE）/ 能量矩阵（FH-MFSK）
4. 均衡后星座（SC-FDE Turbo 末轮 + 4 QPSK 参考点）/ 软 LLR 直方图（FH-MFSK）
5. 信道(时域) — stem |h_tap| vs 抽头索引
6. 信道(频域) — |H_est(f)|(红) vs |H_true(f)|(蓝虚线)
7. 日志 — 全高 uitextarea

V3.0 新增功能：
- **解码历史**：最多 20 条历史记录（带时间戳），RX 面板下拉切换，回看任意一帧的频谱/星座/信道
- **TX 信号信息面板**：每次 Transmit 后显示时长/带宽/样本数/码率/信道预设
- **音频监听**：Mon 按钮实时播放接收通带信号（48kHz，fc=12kHz，人耳可听）
- **Bypass RF 调试开关**：跳过上下变频，复基带直通 + 复 AWGN

实现要点：
- FIFO 8 秒 ring buffer；TX 切片入队，timer 每 100ms pop 一片
- 监听模式下 chunk = tick 周期（1:1 实时），非监听 50ms（2x 加速）
- scheme 切换通过 `Visible='off'` 动态显示对应参数行
- 关闭窗口时 CloseRequestFcn 停掉并删除 timer，防止悬挂

### P3.1 调试记录（2026-04-16 修复）

| 问题 | 根因 | 修复 |
|------|------|------|
| SC-FDE BER≈50% | 短文本 + 零填充 → QPSK 符号无多样性 → GAMP 训练退化 | 零填充改随机填充（seed=42） |
| 通带路径 BER≈50% 但 bypass 正常 | σ²_bb = 8·σ²_pb·BW/fs 公式错写为 4x | 修正因子 4→8 |
| NV 实测覆盖 on_transmit 精确值 | try_decode 用信号前噪声段（默认底噪 0.05）覆盖了正确值 | 仅兜底使用实测值，优先用 on_transmit 设定 |
