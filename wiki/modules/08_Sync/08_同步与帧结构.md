## 功能定位

- **发射端**：生成同步序列，与数据组装成完整发射帧
- **接收端**：
  1. 粗同步：匹配滤波检测帧起点
  2. CFO粗估计：载波频偏估计
  3. 细定时：精确符号定时
  4. 帧拆解：分离导频、数据、保护间隔

## 各体制帧结构

### SC-TDE 帧
```
[LFM前导] [guard] [训练序列(信道估计)] [数据符号] [guard]
```
- LFM用于粗同步和粗多普勒估计
- 训练序列用于LS信道估计
- 无CP，纯时域处理

### SC-FDE 帧
```
[LFM前导] [guard] [data+CP块1] [data+CP块2] ... [guard] [LFM后导]
```
- 前+后导码用于多普勒估计（复自相关幅相联合）
- 每个数据块独立加CP
- 后导码额外占帧开销

### OFDM 帧
```
[ZC双重前导(Schmidl-Cox)] [guard] [OFDM符号1] [OFDM符号2] ...
```
- ZC双重前导：两段相同序列，利用半符号相关进行Schmidl-Cox同步
- 每个OFDM符号含自己的CP

### OTFS 帧
```
[HFM前导(多普勒不变)] [guard] [DD域数据(含整帧CP)]
```
- **HFM前导**：双曲调频，多普勒不变性使得粗同步更鲁棒
- 整帧CP覆盖所有时延展宽
- DD域数据含嵌入导频+保护区

## CFO估计方法

| 方法 | 原理 | 适用场景 |
|---|---|---|
| 相关法 | 前导序列相位差 | 通用，精度中等 |
| Schmidl-Cox | 双重前导半符号自相关 | OFDM标配 |
| CP法 | CP与数据尾部相关 | OFDM在线跟踪 |

## 接口约定

```matlab
% 同步序列生成
lfm_signal = gen_lfm(f0, f1, duration, fs)
hfm_signal = gen_hfm(f0, f1, duration, fs)
zc_seq = gen_zc_seq(N, root)
barker = gen_barker(length)

% 粗同步
[start_idx, peak_val] = sync_detect(rx_signal, ref_seq, threshold)

% CFO估计
cfo_hz = cfo_estimate(rx_signal, method, params)
%   method: 'correlate' | 'schmidl_cox' | 'cp'

% 帧组装（以SC-TDE为例）
tx_frame = frame_assemble_sctde(data, training_seq, lfm_preamble, params)

% 帧解析
[data, training, timing_info] = frame_parse_sctde(rx_signal, params)
```

## 与其他模块的关系

```
发射：[07 导频插入] → [08 同步序列+帧组装] → [09 脉冲成形/上变频]
接收：[09' 匹配滤波] → [08' 同步检测+帧解析] → [11 阵列(可选)] → [10-1 粗多普勒]
```

- **上游**（发射）：接收模块07输出的带导频数据
- **下游**（发射）：完整帧送入模块09成形/上变频
- **接收端**：同步检测是接收链的第一个关键步骤
- 粗同步精度直接影响后续所有模块的性能
- CP的插入/移除由模块06处理，模块08只负责帧级结构

---

## 备注

- LFM是水声通信最常用的前导序列（大时宽带宽积 → 高处理增益）
- HFM的多普勒不变性对高速运动平台特别重要
- Schmidl-Cox方法在OFDM中几乎是标配
- 同步失败 → 整帧丢失，是水声通信可靠性的关键环节
- 细定时TED通常在均衡器内部或均衡后执行
