# 模块06 — 多载波/多域变换+CP (MultiCarrier)

> 文件夹：`06_MultiCarrier/src/Matlab/`
> 框架位置：发射端第6步(可选) / 接收端去CP+逆变换
> OFDM / SC-FDE / OTFS 体制使用；SC-TDE / DSSS / FH-MFSK 跳过

---

## 功能定位

- **发射端**：将频域/DD域符号变换到时域，并插入循环前缀(CP)防止符号间干扰
- **接收端**：去除CP，执行逆变换回到频域/DD域，为均衡器提供输入

## 方案详情

### CP-OFDM
- 发射：频域符号 → IFFT → 加CP(复制尾部到前面)
- 接收：去CP → FFT → 频域单抽头均衡
- CP长度 ≥ 最大多径时延展宽
- 每个OFDM符号独立加CP

### ZP-OFDM
- 用零填充(ZP)代替CP
- 接收端需overlap-add处理
- 避免CP的功率浪费

### SC-FDE
- 发射：时域数据分块 + 每块加CP
- 接收：去CP → 分块FFT → 频域MMSE均衡 → IFFT回时域
- 与OFDM共享频域均衡优势，但PAPR更低

### OTFS-DFT法
- 发射：DD域符号 → ISFFT(逆辛普莱克有限傅里叶变换) → Heisenberg变换 → 时域 + 整帧CP
- 接收：去整帧CP → Wigner变换 → SFFT → DD域
- 等效于先OFDM调制再加DD域预编码

### OTFS-Zak法
- 等效2D-IFFT实现
- 数学上与DFT法等价

## CP设计要点

| 体制 | CP类型 | CP长度要求 |
|---|---|---|
| OFDM | 每符号CP | ≥ τ_max · fs |
| SC-FDE | 每块CP | ≥ τ_max · fs |
| OTFS | **整帧CP** | ≥ τ_max · fs，覆盖所有时延 |

## 接口约定

```matlab
% OFDM
tx_signal = ofdm_modulate(freq_symbols, N_fft, cp_len)
freq_symbols = ofdm_demodulate(rx_signal, N_fft, cp_len)

% SC-FDE
tx_blocks = scfde_add_cp(data_symbols, block_size, cp_len)
freq_blocks = scfde_remove_cp(rx_signal, block_size, cp_len)

% OTFS
tx_signal = otfs_modulate(dd_symbols, N, M, cp_len, method)
%   method: 'dft' 或 'zak'
dd_symbols = otfs_demodulate(rx_signal, N, M, cp_len, method)

% PAPR
papr_dB = papr_calculate(signal)
clipped = papr_clip(signal, threshold_dB, method)
```

## 各体制使用情况

| 体制 | 模块6使用 | 变换方式 |
|---|:---:|---|
| SC-TDE | — | 不使用（纯时域） |
| SC-FDE | 分块CP | scfde_add_cp / scfde_remove_cp |
| DSSS | — | 不使用 |
| **OFDM** | **✓** | IFFT/FFT + CP |
| **OTFS** | **✓** | ISFFT+Heisenberg / Wigner+SFFT |
| FH-MFSK | — | 不使用 |

## 与其他模块的关系

```
发射：[04/05] → [06 多载波变换+CP] → [07 导频插入] → [08 帧组装]
接收：[10-1 粗多普勒] → [06' 去CP+逆变换] → [PTR(可选)] → [07' 均衡]
```

- **上游**：接收模块04/05的符号
- **下游**：变换+CP后送入模块07导频插入
- 接收端06'必须在10-1粗多普勒补偿之后（否则FFT子载波错位）
- OTFS的导频嵌入(`otfs_pilot_embed`)在DD域完成，与模块07协同

---

## 备注

- OFDM的CP开销通常为10%~25%
- SC-FDE的PAPR比OFDM低约2~3dB，水声功放友好
- OTFS整帧CP使得频谱效率在长帧时优于OFDM
- PAPR是水声发射机的重要约束（功放线性度有限）
