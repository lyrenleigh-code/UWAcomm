二、各通信体制的模块选用

单载波时域均衡 (SC-TDE) v6 DFE(31,90)+GAMP+BEM

跳过模块5、6；信道估计: ch_est_gamp（禁止oracle）；接收端PTR空间聚焦 → 时域DFE均衡（iter1: DFE(31,90)+h_est初始化，iter2+: 软ISI消除）；SISO(BCJR)译码器输出外信息；时变场景: 散布导频+BEM每块H_est → 分块LMMSE-IC；Turbo迭代3~6次

TX 1 信源编码→ 2 信道编码→ 3 交织→ 4 符号映射→ 7 导频(训练+散布)→ 8 LFM+帧组装(通带)→ 9 RRC+上变频→ 信道
RX 9' 下变频→ 8' sync_detect→ 11 阵列(可选)→ 10-1 est_doppler+resample(+α)→ 9'' RRC匹配+下采样→ PTR 被动时反转→ 7 ch_est_gamp(h_est)→ 7' DFE(31,90)+h_est(iter1)→ 3' 解交织→ 2' SISO(BCJR)→ 1' 信源解码
↻ Turbo 外信息迭代流程（v6: turbo_equalizer_sctde V8）
信道估计: ch_est_gamp/ch_est_turbo_vamp → h_est（禁止oracle）
iter1: eq_dfe(Nff=31, Nfb=90) + h_est初始化 → soft_demapper → 解交织 → SISO(BCJR)
iter2+: soft_mapper(x̄,σ²x) → 软ISI消除 conv(x̄, h_est) + 单抽头ZF → soft_demapper → 解交织 → SISO(BCJR)
时变场景: 散布导频 → BEM(Q=2⌈fd·T⌉+3) → 每块H_est_bi → iter1用分块LMMSE-IC → iter2+ DD精化
终止条件: 最大迭代次数(3~6次) 或 CRC校验通过
↻ Turbo迭代（3~6次）：7'(DFE/ISI-IC) → 3'(解交织) → 2'(SISO-BCJR) → 3(交织) → 7' 外信息交换Le

单载波频域均衡 (SC-FDE) v6 跨块Turbo+BEM+DD

跳过模块5、6；发射端分块加CP；信道估计: ch_est_gamp（禁止oracle）；跨块Turbo均衡(turbo_equalizer_scfde_crossblock)；时变: 散布导频+BEM每块H_est → 跨块LMMSE-IC + DD信道更新；Turbo迭代3~6次

TX 1 信源编码→ 2 卷积/Turbo编码→ 3 交织→ 4 QPSK/QAM映射→ 7 分块+CP+散布导频→ 8 LFM+帧组装(通带)→ 9 RRC+上变频→ 信道
RX 9' 下变频→ 8' sync_detect→ 11 阵列(可选)→ 10-1 est_doppler+resample(+α)→ 9'' RRC匹配+下采样→ 6' 去CP+分块FFT→ PTR(可选)→ 7 ch_est_gamp(h_est)→ 7' MMSE-IC(H_est_bi)+crossblock→ 3' 解交织→ 2' 跨块BCJR → 1' 信源解码
↻ 跨块Turbo均衡流程（v6: turbo_equalizer_scfde_crossblock）
信道估计: ch_est_gamp → h_est | 时变: BEM重构每块H_est_bi
每块均衡: eq_mmse_ic_fde(H_est_bi) → soft_demapper(Le)
跨块译码: 所有块Le拼接 → 解交织 → 跨块BCJR → Le,dec → 交织 → 分块soft_mapper
DD信道更新: iter2+可用判决数据更新h_est
终止: 3~6次迭代 或 CRC通过
↻ 跨块Turbo（3~6次）：各块7'(eq_mmse_ic_fde) → 拼接 → 3' → 2'(跨块BCJR) → 3 → 分块soft_mapper → 7' + (DD更新H)

直接序列扩频 (DSSS / DS-CDMA)

启用模块5(DSSS直扩)，跳过模块6；无迭代，单次直通

TX 1 信源编码→ 2 信道编码→ 3 交织→ 4 符号映射→ 5 DSSS扩频→ 7 导频插入→ 8 同步+帧组装→ 9 成形/上变频→ 信道
RX 9' 滤波/下变频→ 8' 同步+帧解析→ 11 阵列预处理(可选)→ 10-1 粗多普勒补偿→ 7' 导频提取→估计&均衡→ 10-2 残余相位校正→ 5' 相关解扩→ 4' 符号判决→ 3' 解交织→ 2' 信道解码→ 1' 信源解码

正交频分复用 (OFDM) v6 跨块Turbo+BEM+DD

启用模块6(IFFT/FFT+CP)，跳过模块5；信道估计: ch_est_gamp（禁止oracle）；跨块Turbo均衡(crossblock)；时变: 散布导频+BEM每块H_est → 跨块LMMSE-IC + DD信道更新；Turbo迭代3~6次

TX 1 信源编码→ 2 信道编码→ 3 交织→ 4 QAM映射→ 6 IFFT+CP插入→ 7 散布导频+频域导频→ 8 LFM+帧组装(通带)→ 9 RRC+上变频→ 信道
RX 9' 下变频→ 8' sync_detect→ 11 阵列(可选)→ 10-1 est_doppler+resample(+α)→ 9'' RRC匹配+下采样→ 6' 去CP+FFT→ 7 ch_est_gamp(h_est)→ 7' MMSE-IC(H_est_bi)+crossblock→ 3' 解交织→ 2' 跨块BCJR → 1' 信源解码
↻ 跨块Turbo（3~6次）：各块7'(eq_mmse_ic_fde) → 拼接 → 3' → 2'(跨块BCJR) → 3 → 分块soft_mapper → 7' + (DD更新H)

正交时频空调制 (OTFS)

模块6变体：ISFFT+Heisenberg（发）/ Wigner+SFFT（收）；模块7：DD域嵌入单脉冲导频+保护区；10-1可弱化；MP均衡器必选迭代

TX 1 信源编码→ 2 信道编码→ 3 交织→ 4 QAM映射→ 6 ISFFT+Heisenberg(N×M格点)→ 7 DD域嵌入导频+保护区→ 8 整帧CP+帧组装→ 9 成形/上变频→ 信道
RX 9' 滤波/下变频→ 8' 同步+帧解析→ 11 阵列(可选)→ 10-1 粗对齐(可弱化)→ 6' Wigner+SFFT→DD域→ 7' MP均衡(BP)→ 4' LLR→ 3' 解交织→ 2' 信道解码 → 1' 信源解码
↻ 双层迭代：外层Turbo 3~5次 (7'→3'→2'(SISO)→3→7' 外信息交换) × 内层MP 10~30次(DD域BP)

跳频多频移键控 (FH-MFSK)

模块4变体(MFSK) + 模块5变体(跳频)，跳过模块6；无迭代，单次直通

TX 1 信源编码→ 2 信道编码→ 3 交织→ 4 MFSK映射→ 5 FH跳频→ 7 导频插入→ 8 同步+帧组装→ 9 成形/上变频→ 信道
RX 9' 滤波/下变频→ 8' 同步+帧解析→ 11 阵列预处理(可选)→ 10-1 粗多普勒补偿→ 7' 导频提取→估计&均衡→ 10-2 残余相位校正→ 5' 解跳频→ 4' 能量检测→ 3' 解交织→ 2' 信道解码→ 1' 信源解码

阵列波束形成增强接收（叠加配置）

不独立成体制，而是作为 模块11前处理叠加在上述任意体制之上；输出单路高质量复基带信号，下游流程无需改动

模式A M路ADC采样 R[m,n]→ 时延标定 bf_delay_calibration→ 空时变采样排序+三次样条重建→ 等效采样率 ≈ M·fs→ * 高精度多普勒估计（CRLB降低M²倍）→ 进入标准接收流程
模式B M路ADC采样 R[m,n]→ DAS波束形成（时延对齐+相位补偿）→ SNR提升 ≈ 10·log₁₀(M) dB→ * 标准多普勒估计（低SNR场景适用）→ 进入标准接收流程

V6 信道估计核心规则

禁止oracle作为最终结果
所有体制的信道估计必须调用 ch_est_* 系列函数
ch_est_gamp: 训练序列 → Toeplitz矩阵 → GAMP迭代
ch_est_turbo_vamp: Turbo VAMP算法（精度最优）
静态信道: h_est固定，全块共用
时变信道: CE-BEM基扩展
BEM基数 Q = 2⌈fd·T⌉ + 3
训练+散布导频 → BEM系数c_pq
→ 重构每块 h_est(t_bi)
Oracle(ch_info.h_time)仅用于调试对比，不可作为最终输出

七、框架适配说明与关键调整点

☆ V6 信道估计与帧结构重构

  • 禁止oracle：所有体制的信道估计必须调用 ch_est_gamp 或 ch_est_turbo_vamp，oracle仅用于调试对比
  • 通带实数帧：发射信号为通带实数，可直接DAC输出；帧结构: [LFM|guard|训练|散布导频|数据|...|LFM]
  • 散布导频：数据块间插入已知导频符号，用于时变信道跟踪（BEM基扩展输入）
  • LFM同步：gen_lfm生成通带实数LFM + 功率归一化；接收端 sync_detect(基带LFM参考, 直达径窗口50样本)
  • 多普勒补偿：comp_resample_spline V7 改用正alpha (+α)，修正之前的符号约定错误
  • MATLAB: ch_est_gamp.m、ch_est_turbo_vamp.m、gen_lfm.m、sync_detect.m、comp_resample_spline.m V7

☆ V6 BEM时变信道估计

  • CE-BEM基扩展：Q = 2⌈fd·T⌉ + 3，其中fd为最大多普勒频偏，T为块持续时间
  • 输入：训练序列 + 散布导频位置的已知符号
  • 输出：BEM系数 c_pq，重构每块时变CIR h_est(t_bi)
  • DD精化：Turbo iter2+ 可利用判决反馈数据进一步精化BEM系数
  • 适用：慢变(fd≤1Hz)用Q=5+；快变(fd>1Hz)按公式计算Q
  • 静态(fd=0)：不需要BEM，直接用 ch_est_gamp 输出固定 h_est

☆ Turbo 均衡核心原则与组件 (v5基础 + v6扩展)

  • 外信息原则：Le = Lpost - Lprior，每个SISO模块只输出新增信息，避免自我强化导致发散
  • 迭代环路：7'(SISO均衡) → 3'(解交织) → 2'(SISO-BCJR) → 3(交织) → 7'
  • v6 SC-TDE：iter1改为 DFE(31,90)+h_est初始化 | iter2+ 软ISI消除+单抽头ZF
  • v6 SC-FDE/OFDM：升级为跨块crossblock + DD信道更新
  • v6 时变SC-TDE：转为分块LMMSE-IC + BEM每块H_est（与SC-FDE框架统一）
  • 终止：最大迭代次数(3~6次) 或 CRC通过
  • MATLAB: turbo_equalizer_sctde.m V8、turbo_equalizer_scfde_crossblock.m

⍢ OTFS 必须从 OFDM 拆分的原因

  • 模块6完全不同：OFDM 用 IFFT/FFT，OTFS 用 ISFFT+Heisenberg（发）/ Wigner+SFFT（收）
  • 模块7完全不同：OFDM 频域子载波导频 vs OTFS DD域嵌入单脉冲导频+保护区
  • 模块8有差异：OTFS 用整帧 CP（长度≥τ_max·fs），而非每符号 CP
  • 均衡器完全不同：OTFS 估计稀疏路径 {h_i, l_i, k_i}，使用 MP 均衡器（迭代回环）
  • 多普勒处理方式根本不同：OTFS 在 DD 域直接分辨路径多普勒，模块10-1可弱化
  • 对应 MATLAB 文件：gen_otfs_signal.m、ch_est_otfs.m、mp_detector.m

+ 模块11 阵列预处理的定位与约束

  • 插入位置：8'（同步）之后，10-1（粗多普勒）之前；对下游完全透明
  • 模式A（精度优先）：空时变采样重建，等效采样率提升至 M·fs，CRLB 降低 M² 倍
  • 模式B（SNR优先）：DAS波束形成，SNR 提升约 10log₁₀(M) dB
  • 精度要求：阵元时延标定误差须 < Ts/(2M)，否则等效采样率提升失效
  • MATLAB：bf_delay_calibration.m、bf_nonuniform_resample.m、bf_conventional.m

✎ Turbo迭代环路：7'(SISO-EQ) ⇄ 3'/3(交织) ⇄ 2'(SISO-DEC) (v5基础+v6扩展)

  • 环路结构：7'(SISO均衡,输出Le,eq) → 3'(解交织) → 2'(SISO译码,输出Le,dec) → 3(交织) → 7' | 交织器是环路必要组成部分
  • 10-2位置：残余多普勒补偿在Turbo迭代前一次性完成，不参与迭代环路（v5调整）
  • SC-TDE Turbo(v6)：iter1=DFE(31,90)+h_est初始化 → soft_demapper → 3' → SISO(BCJR) → 3 → soft_mapper → iter2+=ISI消除+ZF → ...
  • SC-FDE/OFDM Turbo(v6)：各块eq_mmse_ic_fde(H_est_bi) → 拼接 → 3' → 跨块BCJR → 3 → 分块soft_mapper → 各块7' + DD更新H
  • OTFS：外层Turbo 3~5次(7'→3'→2'→3→7') × 内层MP(DD域BP) 10~30次
  • DSSS / FH-MFSK：无迭代，单次直通
  • MATLAB调度器：turbo_equalizer_sctde.m V8、turbo_equalizer_scfde_crossblock.m、turbo_equalizer_otfs.m

⚠ SC-TDE 与 SC-FDE 的关键区别 (v6更新)

  • 发射端完全相同（v6: 均为通带实数帧 + 散布导频），区别仅在接收端均衡器结构
  • SC-TDE (v6)：PTR → ch_est_gamp → iter1=DFE(31,90)+h_est初始化，iter2+=软ISI消除+单抽头ZF
  • SC-FDE (v6)：ch_est_gamp → 分块FFT → 跨块crossblock MMSE-IC，支持DD信道更新
  • 时变SC-TDE (v6新增)：转为分块LMMSE-IC + BEM每块H_est，与SC-FDE框架统一
  • SC-FDE 的多普勒估计使用前/后LFM互相关，SC-TDE 同样
  • 参数文件应分开：sys_params_sctde.m vs sys_params_scfde.m
