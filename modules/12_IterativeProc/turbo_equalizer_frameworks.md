## 1 共同基础：Turbo 迭代原理

### 1.1 软信息定义

所有 Turbo 均衡架构共享统一的软信息表示：**对数似然比（LLR）**。

$$L(b) = \ln \frac{P(b=1 \mid y)}{P(b=0 \mid y)}$$

每个模块的输出分解为：

$$L_p = L_e + L_a$$

其中 $L_a$ 为来自对端的先验信息，$L_e$ 为本模块产生的**外信息（Extrinsic Information）**。模块间只传递外信息，避免信息自我强化导致迭代发散。

### 1.2 通用迭代结构

```
初始化: 导频估计信道 Ĥ₀，La,eq = 0

for 迭代轮次 k = 1, 2, ..., K:
    1. SISO 均衡器(y, Ĥ, La,eq) → Le,eq
    2. 去交织:  Le,eq → La,dec
    3. SISO 解码器(La,dec)      → Le,dec, 后验 LLR
    4. 交织:    Le,dec → La,eq
    5. [可选] 软符号映射 → x̄, σ²_x → 更新 Ĥ

输出: 硬判决 x̂
```

### 1.3 迭代调度策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 串行调度（Turbo） | 均衡完整跑 N 轮后再解码，再反馈 | 标准 Turbo，实现简单 |
| 并行调度 | 均衡与解码每轮交替一次 | 低延迟，总轮次少 |
| 置信度门控 | 按软符号可靠度 $\sigma^2_x$ 决定是否更新信道估计 | 迭代信道估计联合设计 |

## 2 时域 Turbo 均衡框架

### 2.1 信道模型

时域离散信道为有限长线性时不变（LTI）卷积：

$$y[n] = \sum_{l=0}^{L-1} h[l] \cdot x[n-l] + w[n]$$

- $L$：信道记忆长度（抽头数）
- $M$：调制阶数（如 QPSK: $M=4$，16-QAM: $M=16$）
- 网格状态数：$M^{L-1}$（指数增长）

### 2.2 核心模块：BCJR/MAP 均衡器

#### 网格（Trellis）构造

```
状态: s = (x[n-1], x[n-2], ..., x[n-L+1])，共 M^(L-1) 个
分支: 每个状态在时刻 n 转移至下一状态，对应输入符号 x[n]
分支度量: λ(s'→s) = log p(y[n] | x[n], h) + La,1(x[n])
```

#### 前向-后向递推（BCJR）

```
前向变量:  α_n(s) = Σ_{s'} α_{n-1}(s') · γ_n(s'→s)
后向变量:  β_n(s) = Σ_{s'} β_{n+1}(s') · γ_{n+1}(s→s')
后验 LLR:  L_p(x[n]) = log[ Σ_{(s'→s):x=+1} α·γ·β / Σ_{(s'→s):x=-1} α·γ·β ]
外信息:    Le,1(x[n]) = Lp(x[n]) - La,1(x[n])
```

#### 先验信息注入方式

解码器反馈的 $L_{a,1}[n]$ 直接修改每条网格分支的转移概率：

$$\gamma_n(s' \to s) \propto p(y[n] \mid x[n]) \cdot \exp\!\left(\frac{x[n] \cdot L_{a,1}[n]}{2}\right)$$

ISI 抑制隐式内嵌于最优 MAP 估计中。

### 2.3 复杂度与降低方案

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| 精确 BCJR | $O(N \cdot M^{L-1})$ | 仅适合 $L \leq 5$ |
| RSSE（状态削减） | $O(N \cdot M^{L_r})$，$L_r < L-1$ | 牺牲性能换复杂度 |
| M-BCJR | $O(N \cdot M_s)$，$M_s$ 固定 | 保留最优 $M_s$ 条路径 |
| SOVA | $O(N \cdot M^{L-1})$，常数小 | 软输出维特比，近似 |

### 2.4 开发要点

- **适用信道**：短记忆（$L \leq 6$），无循环前缀要求，如磁记录、DSL 短环路、水下短距通信。
- **交织器设计**：随机交织器打散突发错误，长度建议 $\geq 1000$ 比特。
- **收敛分析**：使用 EXIT 图（外信息转移图），均衡器与解码器转移曲线之间的"隧道"宽度决定收敛速度。
- **迭代次数**：通常 3～5 轮，可用 CRC 提前停止。

## 3 频域 Turbo 均衡框架

### 3.1 信道模型

引入循环前缀（CP）后，线性卷积变为**循环卷积**，频域表示为逐子载波乘法：

$$Y[k] = H[k] \cdot X[k] + W[k], \quad k = 0, 1, \ldots, N-1$$

其中 $H[k] = \text{DFT}\{h[l]\}$ 为信道频率响应。

### 3.2 核心模块：MMSE-FDE + 软干扰消除

#### 迭代 MMSE 均衡器权重

利用软符号 $\bar{x}[n]$（来自上一轮解码器）和方差 $\sigma^2_x[k]$（残余干扰），逐子载波计算：

$$W[k] = \frac{H^*[k]}{\lvert H[k] \rvert^2 + \sigma^2_w / \sigma^2_x[k]}$$

#### 干扰重建与消除

```
x̄[n]    —— 解码器后验软符号（时域）
X̄[k]    = FFT{x̄[n]}
Ī[k]     = H[k] · X̄[k]              （已知干扰的频域表示）
Ỹ[k]    = Y[k] - (1 - W[k]H[k]) · X̄[k]   （消除后等效接收）
X̃[k]    = W[k] · Ỹ[k]              （均衡输出）
```

#### 等效 AWGN 近似

经 IFFT 后，时域均衡输出近似为：

$$\tilde{x}[n] = \mu \cdot x[n] + \tilde{w}[n]$$

其中有效增益 $\mu = \frac{1}{N}\sum_k W[k]H[k]$，残余噪声近似高斯。于是 LLR 计算退化为标准 AWGN 软解映射：

$$L_{e,1}[n] = \frac{4\mu}{\sigma^2_{\tilde{w}}} \cdot \tilde{x}[n] - L_{a,1}[n]$$

这一近似在块长 $N$ 较大时（由中心极限定理）越来越精确。

### 3.3 迭代流程

```
初始化: x̄[n] = 0, σ²_x = 1（无先验）

for k = 1, ..., K:
    1. FFT: y[n] → Y[k]
    2. FFT: x̄[n] → X̄[k]，计算 Ī[k]
    3. 更新 MMSE 权重 W[k]（用当前 σ²_x[k]）
    4. X̃[k] = W[k](Y[k] - Ī[k])
    5. IFFT: X̃[k] → x̃[n]
    6. 计算等效噪声方差 σ²_ñ
    7. LLR 计算 → Le,1[n]
    8. 去交织 → 解码器 → Le,2，后验 LLR
    9. 软符号映射: x̄[n] = E[x | 后验 LLR]，更新 σ²_x[n]
    10. 交织 → La,1
```

### 3.4 复杂度

| 操作 | 复杂度 |
|------|--------|
| FFT/IFFT | $O(N \log N)$ |
| 逐子载波 MMSE | $O(N)$ |
| 总体（每轮） | $O(N \log N)$，与信道记忆 $L$ 无关 |

### 3.5 开发要点

- **适用场景**：长记忆宽带信道（水声、SC-FDMA）、块传输系统、已有 CP 结构。
- **CP 长度**：必须 $\geq L-1$，否则循环卷积假设不成立。
- **阻尼因子**：早期迭代软符号不可靠，建议引入阻尼 $\bar{x}^{(k)} = \alpha \bar{x}^{(k)} + (1-\alpha)\bar{x}^{(k-1)}$，$\alpha \in [0.5, 0.8]$。
- **OFDM 与 SC-FDE 的区别**：OFDM 每子载波独立传符号（无 ISI，无需均衡），SC-FDE 块内符号存在 ISI，需上述迭代均衡。

## 4 OTFS 时延-多普勒域 Turbo 均衡框架

### 4.1 信道模型

OTFS 将信道建模于时延-多普勒（DD）域。对于 $P$ 条散射路径，DD 域输入输出关系为：

$$y[k,l] = \sum_{i=1}^{P} h_i \cdot x\!\left[(k - k_i)_N,\,(l - l_i)_M\right] \cdot e^{j2\pi k_i (l - l_i)/(NM)} + w[k,l]$$

- $k$：多普勒轴，$k \in \{0, \ldots, N-1\}$
- $l$：时延轴，$l \in \{0, \ldots, M-1\}$
- $(k_i, l_i, h_i)$：第 $i$ 条路径的整数多普勒、整数时延、复增益
- 总符号数：$NM$；每个接收符号仅受 **P 个发送符号**影响（稀疏性）

本质是 $NM \times NM$ 的**稀疏矩阵**乘法，边数约为 $P \cdot NM \ll (NM)^2$。

### 4.2 调制/解调流程

```
发送端:
  x[k,l]（DD 域）→ ISFFT → X[n,m]（TF 域）→ Heisenberg 变换 → s(t)（时域）

接收端:
  r(t) → Wigner 变换 → Y[n,m]（TF 域）→ SFFT → y[k,l]（DD 域）
```

实现中通常等价于：发送端做 ISFFT 后加 CP，接收端去 CP 后做 SFFT。

### 4.3 核心模块：DD 域因子图均衡器

#### 因子图构造

```
变量节点: x[k,l]，共 NM 个（代表发送符号）
因子节点: y[k,l]，共 NM 个（代表接收观测）
边: 若路径 i 使 x[(k-kᵢ)_N, (l-lᵢ)_M] 影响 y[k,l]，则连边，边权为 hᵢ·e^{jφ}
图的度数: 每个因子节点度为 P，每个变量节点度为 P（稀疏）
```

#### 消息传递更新（高斯近似 / GAMP）

**变量节点 → 因子节点**（上行消息）：

$$\mu_{x \to f}^{(t)}[k,l] = \mathcal{N}\!\left(\bar{x}[k,l],\, \sigma^2_x[k,l]\right)$$

其中 $\bar{x}, \sigma^2_x$ 由解码器反馈的先验 LLR 通过软符号映射得到。

**因子节点 → 变量节点**（下行消息，软干扰消除）：

$$\tilde{y}_i[k,l] = y[k,l] - \sum_{j \neq i} h_j \cdot \bar{x}\!\left[(k-k_j)_N,(l-l_j)_M\right] \cdot e^{j\phi_j}$$

$$\tilde{\sigma}^2_i[k,l] = \sigma^2_w + \sum_{j \neq i} |h_j|^2 \sigma^2_{x,j}$$

等效为 AWGN 信道后计算外信息 LLR：

$$L_{e,\text{eq}}[k,l] = \frac{2 h_i^* \tilde{y}_i[k,l]}{\tilde{\sigma}^2_i[k,l]} - L_{a}[k,l]$$

#### 迭代流程

```
初始化: x̄[k,l] = 0, σ²_x = 1

for k = 1, ..., K_outer:           # Turbo 外迭代
    for t = 1, ..., T_mp:          # MP 内迭代
        for 每个因子节点 y[k,l]:
            消除其他 P-1 条路径干扰 → ỹᵢ, σ̃²ᵢ
            计算外信息 Le,eq[k,l]
        for 每个变量节点 x[k,l]:
            汇聚所有因子节点消息 → 更新后验均值 x̄, 方差 σ²_x
    
    去交织 → 解码器 → 后验 LLR
    软符号映射 → 更新 x̄[k,l], σ²_x[k,l]
    交织 → La[k,l]
    [可选] 用 x̄ 更新信道估计 (k̂ᵢ, l̂ᵢ, ĥᵢ)
```

### 4.4 分数时延/多普勒处理

实际路径参数 $(\tau_i, \nu_i)$ 不落在整数格点时，稀疏性被破坏。主要应对方案：

| 方案 | 思路 | 代价 |
|------|------|------|
| 保护间隔（DD Guard） | 在 DD 格点周围留空白，限制泄漏范围 | 降低频谱效率 |
| 基扩展模型（BEM） | 用 $Q$ 个基函数近似分数多普勒，展开因子图边权 | 因子图边数增至 $P \cdot Q$ |
| 过采样 + 插值 | 增大 $N, M$ 使格点更密，分数偏移变小 | 复杂度 $O(NM \log NM)$ 增大 |

### 4.5 三种主流均衡算法对比

| 算法 | 复杂度 | 精确性 | 收敛保证 | 分数多普勒 |
|------|--------|--------|----------|------------|
| MP（消息传递） | $O(P \cdot NM \cdot I_{mp})$ | 高（稀疏图近似最优） | 可能振荡 | 需 BEM 改造 |
| MMSE 线性均衡 | $O((NM)^3)$ 或结构加速 $O(NM \log NM)$ | 次优（高斯近似） | 保证 | 直接处理 |
| EP（期望传播） | $O(P \cdot NM \cdot I_{ep})$，常数略大 | 介于 MP 与 MMSE | 阻尼 EP 可保证 | 可处理 |

### 4.6 开发要点

- **稀疏性是核心假设**：设计时应验证路径数 $P$ 是否远小于 $NM$，否则 MP 退化。
- **相位旋转项** $e^{j2\pi k_i(l-l_i)/(NM)}$ 必须在因子图边权中精确体现，忽略会导致明显性能损失。
- **帧尺寸选择**：$N$（多普勒分辨率 $= \Delta\nu = 1/(NT_s)$），$M$（时延分辨率 $= \Delta\tau = T_s/M$），需与最大多普勒和最大时延匹配。
- **与 ISAC 联合设计**：DD 域信道参数 $(k_i, l_i, h_i)$ 直接对应物理散射体的距离-速度，天然支持感知通信一体化。

## 5 迭代信道估计联合设计

### 5.1 软导频机制

解码器的后验 LLR 产生**软符号**，作为可靠度随迭代提升的虚拟导频：

$$\bar{x} = \sum_{s \in \mathcal{A}} s \cdot P(x = s \mid y, L_{e,2})$$

$$\sigma^2_x = \sum_{s \in \mathcal{A}} |s|^2 P(x=s) - |\bar{x}|^2$$

### 5.2 三类框架中的信道估计方法

| 框架 | 推荐估计方法 | 更新对象 | 备注 |
|------|-------------|----------|------|
| 时域 | 加权 LS / Kalman 滤波 | 时域抽头 $h[l]$ | 时变信道用 AR 模型建模多普勒 |
| 频域 | 软导频 MMSE / EM | 频域响应 $H[k]$ 或抽头 $h[l]$ | 利用频域平滑先验（如 DFT 域稀疏） |
| OTFS | SBL / EM + 稀疏先验 | DD 域参数 $(k_i, l_i, h_i)$ | 稀疏贝叶斯学习天然匹配 DD 稀疏性 |

### 5.3 调度策略

```
策略 A（外循环更新）:
    内层: 均衡/解码迭代 N 轮
    外层: 用稳定的软符号更新 Ĥ，再重跑内层
    优点: 信道估计质量高；缺点: 总轮次多

策略 B（每轮更新）:
    每次均衡-解码交替后立即更新 Ĥ
    优点: 总轮次少；缺点: 早期软符号不可靠，误差传播风险

工程推荐: 置信度门控
    if mean(1 - tanh(|Le|/2)) < threshold:
        跳过本轮信道更新
    else:
        用可靠位置的软符号更新 Ĥ
```

### 5.4 防误差传播措施

- **可靠度加权**：用 $w_n = \tanh^2(|L_{e,2}[n]|/2)$ 对软符号加权，低可靠度位置权重趋零。
- **导频锚定**：迭代中始终保留导频符号的贡献，防止信道估计漂移。
- **前几轮冻结**：前 1~2 轮不更新信道，待软符号质量达到门限再开放。

## 6 三类框架横向对比

| 维度 | 时域 Turbo 均衡 | 频域 Turbo 均衡 | OTFS DD 域 Turbo 均衡 |
|------|----------------|----------------|----------------------|
| **信道表示** | 时域 1D 卷积，稠密时变 | 频域逐子载波乘法，对角矩阵 | DD 域 2D 稀疏卷积，准静态 |
| **均衡器类型** | BCJR/MAP，网格递推 | MMSE-FDE，线性滤波 | MP/GAMP，因子图消息传递 |
| **软信息注入** | 修改网格分支转移概率 | 干扰重建 + 更新 MMSE 权重 | 变量节点消息，逐路径干扰消除 |
| **均衡复杂度** | $O(N \cdot M^L)$，指数 | $O(N \log N)$，与 $L$ 无关 | $O(P \cdot NM \cdot I)$，与路径数线性 |
| **最优性** | 精确 MAP 最优 | 高斯近似，次优 | 稀疏图近似最优（MP）|
| **时变信道** | 需逐符号更新网格 | ICI 严重，性能下降 | 天然处理，多普勒即信道参数 |
| **CP/保护间隔** | 不需要 | 需要 CP，长度 $\geq L-1$ | 需要 DD 保护区或 CP |
| **信道估计** | 时域 Kalman/LS | 频域 EM/MMSE | SBL + 稀疏先验，EM |
| **典型场景** | 磁记录、DSL、短记忆无线 | SC-FDMA、水声宽带、4G/5G | 高速移动、LEO 卫星、V2X |
| **实现难点** | 长记忆指数爆炸 | 等效 AWGN 近似在短块退化 | 分数多普勒破坏稀疏性 |

## 7 选型指南

### 决策流程

```
信道记忆是否很长（L > 6）?
├── 否 → 时域 Turbo 均衡（精确 MAP，复杂度可控）
└── 是 → 系统是否已有 CP 结构（OFDM / SC-FDMA）?
          ├── 是 → 频域 Turbo 均衡（FFT 加速，O(N log N)）
          └── 否 → 信道是否高速时变（高多普勒）?
                    ├── 是 → OTFS DD 域 Turbo 均衡
                    └── 否 → 频域（加 CP）或时域降阶（M-BCJR）
```

### 混合架构

| 场景 | 推荐 |
|------|------|
| 残余 ISI（频域均衡后） | 频域粗均衡 + 时域 MAP 精处理 |
| OFDM 时变信道（ICI） | 子带分组 + 时域局部均衡 |
| OTFS 长帧 + 低复杂度需求 | MMSE 结构化矩阵快速求逆（块循环矩阵 FFT 分解）|
| ISAC 一体化 | OTFS DD 域联合均衡+感知（信道参数即物理参数）|
